<objective>
## Goal
Analyser le dataset médical hérité, le nettoyer, et produire un rapport de qualité des données prêt pour le fine-tuning LoRA.

## Purpose
Le fine-tuning LoRA dépend directement de la qualité des données. Un dataset mal nettoyé (doublons, champs manquants, format incohérent) dégrade les performances du modèle. Ce plan prépare les données avant le lancement sur Colab.

## Output
- `scripts/clean_dataset.py` : script de nettoyage reproductible
- `data/medical_dataset_clean.json` : dataset nettoyé
- `reports/data_quality_report.md` : rapport qualité complet
</objective>

<task type="auto">
  <name>Task 1: Analyser le dataset médical et identifier les anomalies</name>
  <files>scripts/clean_dataset.py, reports/data_quality_report.md</files>
  <action>
    Créer scripts/clean_dataset.py.
    Charger medical_dataset/ : détecter si c'est un fichier JSON unique ou plusieurs fichiers.
    Analyser la structure : fields présents (ex: "instruction", "input", "output" ou "question", "answer" ou "messages"), types de valeurs, longueurs.
    Compter : nb total entrées, nb entrées avec champs manquants, nb doublons (hash sur contenu), longueur min/max/moyenne des textes.
    Identifier le format de conversation attendu par LoRA (chat format : [{"role":"user","content":"..."}, {"role":"assistant","content":"..."}]).

    - Section "Dataset source" : fichier(s), nb entrées brutes, champs détectés
    - Section "Anomalies détectées" : entrées vides, champs manquants, doublons, textes trop courts (<10 chars)
    - Section "3 exemples" : 3 entrées représentatives du dataset
    - Section "Évaluation qualité" : Bon (>90% entrées valides) / Moyen (70-90%) / Insuffisant (<70%)
  </action>
  <verify>python3 scripts/clean_dataset.py s'exécute sans erreur et reports/data_quality_report.md est créé</verify>

    Afficher : "Dataset prêt : X entrées pour fine-tuning LoRA"
  </action>
