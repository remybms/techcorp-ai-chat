<objective>
## Goal
Installer Ollama, charger Phi-3.5-Financial, et valider que l'API répond correctement sur localhost:11434.

## Purpose
Le serveur d'inférence est le fondement de tout le projet. Sans lui opérationnel, l'interface web ne peut pas fonctionner. Ollama est choisi pour sa simplicité (30min setup vs 2-3h Triton).

## Output
- Ollama installé et démarré
- Modèle Phi-3.5-Financial chargé et répondant
- `setup/ollama_setup.sh` : script reproductible de setup
- Validation API documentée dans l'audit
</objective>

<acceptance_criteria>

## AC-1: Ollama installé et démarré
```gherkin
Given une machine sans Ollama
When on exécute le script d'installation
Then ollama serve tourne sur localhost:11434 et répond à curl http://localhost:11434
```

## AC-2: Phi-3.5-Financial chargé
```gherkin
Given Ollama opérationnel
When on lance ollama run ou ollama pull avec le modèle financial
Then le modèle est disponible dans ollama list et répond à une question financière simple
```

## AC-3: API REST fonctionnelle
```gherkin
Given le modèle chargé dans Ollama
When on envoie POST http://localhost:11434/api/generate avec un prompt financier
Then on reçoit une réponse JSON avec le champ "response" non vide en moins de 30s
```
